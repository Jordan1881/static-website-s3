Static Website on AWS S3

Deployed automatically via GitHub Actions

CI/CD Pipeline — Live

What We Built

1

GitHub Repository

Created a public repo (static-website-s3) with HTML files and initialized with a README.

2

Static Website Files

Wrote index.html and about.html — the content you're looking at right now.

3

AWS S3 Bucket

Created an S3 bucket with public access enabled and static website hosting turned on, with index.html as the entry point.

4

Bucket Policy

Added a public-read bucket policy so anyone can load the site files via the S3 website endpoint.

5

IAM User & Permissions

Created a github-actions-user in IAM with a scoped inline policy granting only the S3 actions needed to deploy: PutObject, GetObject, DeleteObject, ListBucket.

6

GitHub Secrets

Stored AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_S3_BUCKET, and AWS_REGION as encrypted repository secrets — never in code.

7

GitHub Actions Workflow

Added .github/workflows/deploy.yml — on every push to main, it checks out the code, authenticates to AWS, and runs aws s3 sync to push all files to the bucket.
